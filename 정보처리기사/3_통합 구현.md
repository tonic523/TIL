# 통합 구현
    서로 다른 시스템간의 원활한 통신을 위한 연계 모듈 구현을 목표로 하고 있습니다.

## 1. 연계 데이터 구성

### 연계 요구사항 분석
- 분석기법
    - 인터뷰, 설문지, 델파이 기법, 브레인스토밍

### 연계 시스템 구성
- 송신 시스템, 수신 시스템, 중계 서버

## 2. 연계 메커니즘 구성
- 개념
    - 응용 소프트웨어와 연계 대상 모듈 간의 데이터 연계 시 요구사항을 고려한 연계방법과 주기를 설계하기 위한 메커니즘

| 연계 방식 | 기술 | 장점 | 단점 |
|:---:|:---:|:---|:---|
|직접 연계|DB링크, DB연결, API, JDBC|연계 및 통합 구현이 단순하며 용이함, 개발 소요 비용 및 기간이 짧음|시스템 간의 결합도가 높아 시스템 변경에 민감, 암.복호화 처리 불가|
|간접 연계|EAI, ESB/Web Service, 소켓|서로 상이한 네트워크, 프로토콜 연계 및 통합 가능, 인터페이스 변경 시에도 장애나 오류 없이 서비스 가능|개발 및 적용을 위한 테스트 기간이 상대적 장기간 소요|

- 로그 기록
    - 송신 시스템과 수신 시스템에서 처리되는 일련의 과정에 대한 결과를 로그 테이블에 기록 및 관리한다.

## 3. 내외부 연계 모듈 구현
- EAI
    - 기업에서 윤영되는 서로 다른 플랫폼 및 애플리케이션들 간의 정보 전달, 연계, 통합을 가능하게 해 주는 솔루션이다.
- ESB
    - 기업에서 운영되는 서로 다른 플랫폼 및 애플리케이션들 간을 하나의 시스템으로 운영할 수 있도록 서비스 중심의 통합을 지향하는 아키텍처 또는 기술이다.
    - 버스를 중심으로 각각 프로토콜이 호환이 가능하도록 애플리케이션의 통합을 느슨한 결합 방식으로 지원하는 방식이다.
- EAI/ESB 방식 연계 모듈 환경 구축 절차
    1. 연계 DB 또는 계정 생성
    2. 연계를 위한 테이블 생성
    3. 연계 응용 프로그램 구현

    - 트리거의 작성 사례

        ~~~
        CREATE OR REPLACE TRIGGER 프로그램명
        AFTER INSERT,UPDATE,DELETE
        ON 테이블 명
        FOR EACE ROW 
        DECLARE AFTER_CODE VARCHAR2(5);
        BEGIN
        SELECT ___
        INTO AFTER_CODE
        FROM ___
        WHERE ___
        INSERT INTO 로그 테이블명
        (칼럼명, 칼럼명)
        VALUES ___;
        INSERT INTO 연계 테이블명
        (칼럼명, 칼럼명)
        VALUES ___;
        EXCEPTION___
        END;
        ~~~

- 웹 서비스 방식
    - 네트워크에 분산된 정보를 서비스 형태로 개방하여 표준화된 방식으로 공유하는 기술로써 서비스 지향 아키텍처 개념을 실현하는 대표적인 기술
    - 종류: SOAP, UDDI, WSDL

- SOAP
    - HTTP,HTTPS 등을 사용하여 XML 기반의 메시지를 네트워크 상태에서 교환하는 프로토콜
    - HTTP 프로토콜 상에 SOAP Envelope, 헤더, 바디 등이 추가된 XML 문서로 기본적인 송수신은 HTTP로 수행
- WSDL
    - 웹 서비스명, 제공 위치, 메시지 포맷, 프로토콜 정보 등 웹 서비스에 대한 상세 정보를 기술한 파일로 XML 형식으로 구현
- UDDI
    - 웹 서비스에 대한 정보인 WSDL을 등록하고 검색하기 위한 저장소로 공개적으로 접근, 검색이 가능한 레지스트리
